Research Proposal Review: Methodology, Data Collection, and Skill Development

1. Which of the methods described in this week's reading would you think would suit your purpose?

In the context of my project, the most relevant methodological category is descriptive research. This approach aims to expand understanding by collecting and analysing data that describes the current state of a system, in this case, CHO and iPSC bioprocesses. Descriptive research helps to map out how certain process parameters behave across different scales and cell types, which is essential before moving into model training or prediction. It allows for identifying patterns, validating observations from literature or lab work, and laying the groundwork for more advanced modelling. While the project ultimately incorporates predictive modelling, it begins with a descriptive phase focused on examining and understanding the available data to ensure that the model is built on reliable and meaningful inputs.

2. Which data collection methods would you consider using?

Data will be collected from two main sources. CHO datasets will be obtained from publicly available databases, literature, and existing demo runs, covering a wide range of process parameters and bioreactor scales. In parallel, small-scale iPSC process data will be generated in the lab and supplemented with selected case studies from published research or collaborators. Additional information will be extracted from peer-reviewed studies to ensure the dataset includes the most relevant and reliable variables.

3. Which required skills will you need to have or develop for the chosen project?

To complete this project, I will need to build on several technical and domain-specific skills. This includes developing a deeper understanding of machine learning, especially supervised learning techniques and transfer learning, as well as gaining experience in Python programming using libraries such as scikit-learn, and SHAP for model interpretation. I will also need to refine my data preprocessing skills to clean, standardise, and align complex time-series data from different sources. Alongside this, I aim to strengthen my understanding of bioprocesses, particularly STR scale-up and iPSC culture. Finally, strong scientific communication and the ability to critically interpret literature will be essential for reporting the findings and integrating knowledge from multiple domains.

Research Proposal Outline

Project Title:

A Data-Driven Perspective on Predictive Bioreactor Scale-Up: From Industrial CHO Models to iPSC Applications

Research Question:

Can knowledge from CHO bioprocess data be transferred via ML to support predictive scale-up of iPSCs with minimal experimental data?

1. Introduction

  • Overview of bioprocess scale-up and its challenges
  • Importance of STRs in industrial and academic bioproduction
  • Shift from CHO to iPSC systems in biotechnology
  • Motivation for integrating machine learning into scale-up strategies
  • Research gap: limited ML applications in iPSC bioprocessing

2. Aim and Objectives

Aim:

To develop and test an ML model that uses CHO bioprocess data to predict iPSC performance at larger scales through transfer learning.

Objectives:

  • Collect and preprocess CHO bioprocess data
  • Generate and compile small-scale iPSC data
  • Extract additional iPSC-related data from literature
  • Train baseline ML model on CHO data
  • Fine-tune model with iPSC data
  • Validate model on larger-scale iPSC runs
  • Analyse performance and feature importance (e.g., with SHAP)

3. Background and Literature Review

  • CHO systems: standard model in industrial bioprocessing
  • iPSCs: emerging but fragile, with growing industrial relevance
  • Existing ML models for bioprocessing (CHO- and bacteria-based)
  • Transfer learning: concept, use in biology, and relevance here
  • Limitations of current approaches and need for cross-cell type modelling

4. Methodology

Data Collection:

  • CHO: literature, databases, demo runs (parameters: VCD, viability, P/V, volume...)
  • iPSC: small-scale lab data (500 mL–5 L), supplemented with public case data

Preprocessing:

  • Data cleaning, unit standardisation, handling missing values, time-series alignment

Model Development:

  • ML model (e.g., neural network) trained on CHO data
  • Fine-tuning with iPSC data to increase specificity

Validation:

  • Predict VCD for larger-scale iPSC runs
  • Evaluate using RMSE, R²
  • SHAP analysis for interpretability

Tools and Libraries:

  • Python, scikit-learn, SHAP

5. Expected Outcomes

  • Proof-of-concept: CHO-trained model adapted for iPSC prediction
  • Cost- and time-efficient early-stage iPSC scale-up strategy
  • Insights into key scale-sensitive variables between CHO and iPSCs
  • ML model artefact, visual outputs, final report and presentation

6. Significance

  • Demonstrates value of transfer learning in bioprocess modelling
  • Bridges two cell systems with differing complexity
  • Reduces experimental burden in resource-limited settings
  • Supports future work in iPSC-based bioproduction

7. Ethical Considerations

  • Transparent use of data with appropriate citations and permissions
  • Model limitations clearly stated to avoid misuse
  • Designed as a supportive tool, not a replacement for experiments
  • Full documentation and reproducibility of results

8. Timeline

  • Months 1–3: Data collection, lab work, preprocessing, literature review
  • Months 3–4: Model training (CHO), fine-tuning with iPSC data
  • Months 4–5: Validation, SHAP analysis, visualisation
  • Months 5–6: Final report and presentation
